## 1.33.6 — Agent Runtime Layer (commit c24b3dd)

New file: `src/sourcecode/mcp/orchestrator.py`

Session state machine:
- States: INIT → CONTEXT_LOADED → STALE_CONTEXT → INCOMPLETE_CONTEXT → TASK_INTENT_DETECTED → READY_FOR_REVIEW

Executable orchestration rules:
- R1: stale cache → prepend get_delta to any tool sequence
- R2: Java + no endpoint index → prepend get_endpoints
- R3: repo > 1000 classes → RIS path flagged as preferred

Intent detection (regex, ~0ms, no LLM):
- 6 intents: pr_review, bug_investigation, feature_implementation, refactor, test_generation, orientation

5 new MCP tools (appear first in tool list):
1. `start_session(repo_path, task_description)` — PRIMARY entry point. Single call replaces all manual tool selection. Returns session_state + recommended_next_action + required_tools_sequence + KPIs.
2. `analyze_task(repo_path, task_description)` — Standalone intent detection. Extracts symptom from bug descriptions. Warns on missing context.
3. `run_pr_review_flow(repo_path, since)` — Auto-chains: get_delta → review_pr_context → get_impact_context (top 3 changed Java classes). Auto-detects merge-base.
4. `run_bug_investigation_flow(repo_path, symptom)` — Auto-chains: fix_bug_context → get_impact_context → get_ir_summary. Extracts suspect class from symptom.
5. `run_feature_flow(repo_path, feature_description)` — Auto-chains: context (RIS fast-path) → get_endpoints → get_delta(HEAD~3) → refactor_context.

KPIs tracked in session_meta on every response:
- `ttfca_ms` — time_to_first_correct_action
- `tools_suggested` — decision reduction (1-3 vs 18 free tools)
- `orchestration_rules_applied` — which rules fired

`get_cold_start_context` docstring updated: "PREFER start_session over this tool."
