## Session 45 — 5-Fix Hardening Audit

**Objective**: Reproduce and fix 5 blocking errors detected in v1.33.14 audit against
keycloak, spring-petclinic, BroadleafCommerce.

### Fix 1 — Score flatten (prepare_context.py)
**Symptom**: All files scored 1.000, exact class matches buried.
**Root cause**: `_boost_totals` tracked visible delta only (0 for files already at 1.0 base).
Files with 3+ path keywords started at score=1.0 cap, leaving no differentiation headroom.
**Fix**: Track `_raw_signals` = uncapped accumulated signal per file. Sort by raw signal.
Files with more evidence (path + content + commit boosts) now rank above equal-capped files.
**Verified**: `NPE in UserProvider findById` → UserProvider.java ranks #2, JpaUserProvider #1.

### Fix 2 — Over-recall (prepare_context.py)
**Symptom**: 94–134 files in keycloak, 101–141 in Broadleaf — too many for agent context.
**Root cause**: Pass 4 + 4c generated too many candidates even with per-keyword caps.
**Fix**: Cap output at 40 files for large repos after final sort (raw signal sort preserves
the most relevant files). `_OUTPUT_CAP = 40`.
**Verified**: keycloak LDAP: 94→40, Broadleaf payment: 109→40. LDAP files still top.

### Fix 3 — Cold-start schema (ris.py)
**Symptom**: MCP `get_cold_start_context` returned `cache_source` at top level without
`_cache` wrapper. Agents reading `_cache.cache_source` got None silently.
**Fix**: Added `_cache` dict to `get_cold_start_context` return value matching CLI schema.
**Verified**: `_cache present: True`, `_cache.cache_source: RIS`.

### Fix 4 — Cold scan performance (file_classifier.py)
**Symptom**: Keycloak cold compact: ~14.3s (was 9s in v1.31.17).
**Root cause**: `_has_any_import()` called `_matched_imports()` internally, then caller
called `_matched_imports()` again for evidence — doubling the work.
Profiler confirmed: 32,016 `_matched_imports` calls for 25,550 classify calls.
**Fix**: Call `_matched_imports` once per category and check the result directly.
**Verified**: 13.5s → 13.0s (0.5s improvement). Not the full 4.5s regression from 1.31.17 —
remaining overhead likely in Java import graph construction added in 1.33.x.

### Fix 5 — Grep injection (prepare_context.py)
**Symptom**: "login fails after password change" → 8 SAML adapter files (wrong).
**Root cause**: "login" in `_FRONTEND_SYMPTOM_MAP` triggered Pass 4b grep for
"authentication" across keycloak → SAML files flooded results → budget trimmer kept 8 wrong.
**Fix**: In large repos, skip Pass 4b for frontend keywords that already have direct
path matches (they're backend terms, not UI terms needing translation).
**Verified**: "login fails after password change" → 40 files: OIDCLoginProtocolService,
ResetCredentialEmail, LoginFormsProvider — correct login/password files.
