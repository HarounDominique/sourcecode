## What Was Done This Session

Performance engineering mission — fixed P-2 (CRITICAL) and P-1 (HIGH) from session 83.

### P-2 FIXED: keycloak spring-audit hang → 21.7s (was 300s+)

**Root cause 1 — catastrophic regex backtracking (PRIMARY):**
`_ANN_WITH_ARGS_RE` in `repository_ir.py` catastrophically backtracks on files with
deeply-nested annotation arguments: `@APIResponse(content=@Content(schema=@Schema(...)))`.
`_normalize_multiline_annotations` merges these multi-line annotations into a single long line
(e.g. 200+ chars), then the regex tries all combinations and never terminates.

**Fix:** Replaced `_ANN_WITH_ARGS_RE.match(stripped)` with new `_parse_annotation_line(stripped)` —
an O(n) paren-counter that never backtracks. Same API: returns `(ann_name, ann_args)`.
Added at line ~358 in `repository_ir.py`.

**Root cause 2 — 5918 files all hitting full extraction:**
Only 303/5918 keycloak files have Spring/JPA/CDI annotations; the other 5615 were wastefully
parsed by `_extract_symbols`.

**Fix:** Pre-scan filter in `build_repo_ir` Pass 1 loop. Tuple `_ANNOTATION_MARKERS` of 30
markers checked with `any(marker in source for marker in _ANNOTATION_MARKERS)` — C-speed
substring scan. Skipped files still register minimal symbols (package + classname) for
same-package type resolution. Filter rate: 94.9%.

**Root cause 3 — O(N²) BFS on large repos:**
`_bfs_reachability` called for every symbol. Fixed with `_BFS_SYMBOL_THRESHOLD = 5000`
in `_assemble`: skip BFS entirely for large repos.

**Regression note:** `@Entity` not in original markers → BUG-5 (Owner.java stereotype)
regressed. Fixed by adding `@Entity`, `@MappedSuperclass`, `@Embeddable`, `@Aspect`,
`@Document`, `@Query`, `@NamedQuery` to `_ANNOTATION_MARKERS`.
