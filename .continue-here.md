### NEW BUG-A FOUND AND FIXED — check_freshness false positive
**Symptom**: check_freshness keycloak+broadleaf → fresh=False even though
current_git_head == ris_git_head.
**Root cause**: ris.py:_has_uncommitted_changes used `git status --porcelain`
which includes untracked files. Both repos have legacy `.sourcecode-cache/`
dir (untracked) → has_uncommitted_changes=True → fresh=False.
**Fix**: Added `--untracked-files=no` to git status. Only staged/unstaged tracked
file changes trigger has_uncommitted_changes=True.
**File**: `src/sourcecode/ris.py:360`
**Commit**: 86486de

### NEW BUG-B FOUND AND FIXED — --changed-only ignored on warm repos
**Symptom**: `sourcecode /path --changed-only` on clean warm repo returned full
cached output instead of {changed_files:[], analysis_scope:"empty"}.
**Root cause**: L2 cache hit at cli.py:1269 returned early BEFORE changed_only
logic at line 1940. Cache hit bypassed all --changed-only handling.
**Fix**: `if _cache_hit_content is not None and not changed_only:` — skip L2
early return when changed_only=True.
**File**: `src/sourcecode/cli.py:1269`
**Commit**: 86486de

### NEW BUG-C FOUND AND FIXED — untracked dirs counted as changed files
**Symptom**: keycloak --changed-only returned full output (not empty) even after
BUG-B fix, because `.sourcecode-cache/` was in _allowed_changed_files.
**Root cause**: `set(_uc.untracked)` included paths with trailing "/" (git
porcelain marks untracked dirs with "/"). Dirs are not source files.
**Fix**: `_uc_files = {p for p in _uc.untracked if not p.endswith("/")}`.
**File**: `src/sourcecode/cli.py:1951`
**Commit**: 86486de
