## Session 40A — Spring DI Fix (commit 12b0ba9 → 1.33.10)

Root cause: `_build_relations()` never emitted `injects` edges for constructor-injected
dependencies. Constructor `SymbolRecord` had `imports_used=[]` hardcoded; `param_types`
populated but never consumed. Modern Spring Boot uses constructor injection — `impact
OwnerRepository` returned empty results.

Fixes in `repository_ir.py`:
- Constructor injection (no `@Autowired`, Spring 4.3+): resolve each `param_type` via
  `import_map`, emit `injects` from `ClassName#<init>` to FQN
- `@Autowired` constructor: same path
- `@RequiredArgsConstructor`: scan source lines for `private final` fields, emit `injects`
- `@AllArgsConstructor`: same scan, all non-static fields
- `_LOMBOK_CTOR_ANNOTATIONS` constant added after `_INJECT_ANNOTATIONS`

10 new tests in `TestSpringDIInjection` (test_repository_ir.py).
Petclinic regression: `impact OwnerRepository` now reaches `OwnerController`.

## Session 40B — Contract/Compat Audit (commit 473dea9 → 1.33.11)

### P1 — fix-bug JSON contract
Warning `[fix-bug] Results are significantly better with --symptom...` was being
mixed into JSON stdout via CliRunner (mix_stderr=True by default in this typer version).
Fix: `isatty()` guard — warning only emits to interactive terminals.
Also applied same guard to impact legacy-swap warning.
`getattr(sys.stderr, "isatty", lambda: False)()` pattern handles FakeStdout objects.

### P2 — impact legacy arg compatibility
Old syntax: `sourcecode impact <path> <target>`. New: `sourcecode impact <target> [path]`.
Fix: if `Path(target).is_dir()` and `path` doesn't resolve as dir → detect legacy order,
transparently swap, emit TTY-only warning.
Error hint updated to explain new syntax when dir lookup fails.

### P3 — fix-bug ranked_files alias
`relevant_files` renamed from `ranked_files` in a previous version.
Fix: emit `ranked_files` alongside `relevant_files` in fix-bug output (backward compat
alias — same content). Pattern matches existing `test_gaps` / `untested_sources` alias.

### P4 — generated_at consistency
Both COMPACT and AGENT fresh runs: `generated_at` always populated (tested).
Cache hits recover from stored `_cache.generated_at` — null only for pre-timestamp
cache entries (intentional, documented via test assertions).

### P5 — review-pr structured error
`no_diff_source` error when no `--since` now includes actionable hints:
`"Common values: --since HEAD~1 | --since origin/main | --since main"`.
Also clarifies staged-changes workflow.

### P6 — mcp list-tools (new command)
`sourcecode mcp list-tools` — lists all MCP tool names + first-line descriptions.
`sourcecode mcp list-tools --json` — returns `[{name, description}, ...]` array.
Backed by `asyncio.run(mcp.list_tools())`. Discovers `explain_context`,
`refactor_context`, and all other hidden tools.

### P7 — MCP version structured JSON
`version_cmd` now outputs structured JSON on non-TTY stdout:
`{"cli_version": "...", "mcp_schema_version": "...", "compatibility_schema_version": "1.0"}`
Human-readable `"sourcecode X.Y.Z"` still shows on interactive terminals.
The registry-wrapped MCP `version()` tool now returns this dict (not a string).

28 new regression tests in `tests/test_bug_fixes_v13311.py`.

<decisions_made>

- `isatty()` is the correct gate for informational warnings in CLI/MCP dual-mode context.
  `getattr(sys.stderr, "isatty", lambda: False)()` pattern is robust against test stubs.
- `ranked_files` stays as permanent alias (not deprecated+removed) — same pattern as
  `test_gaps` / `untested_sources`. Low cost, breaks no consumers.
- `version_cmd` TTY-detection: `getattr(sys.stdout, "isatty", lambda: False)()` —
  FakeStdout objects in tests don't have isatty, so they get JSON (non-TTY behavior).
  This is correct: tests should always get machine-readable output.
- `_finalize_mcp_registry()` in server.py overwrites ALL `@mcp.tool()` decorated
  functions via `globals()[spec.name] = tool_fn`. P7 fix therefore lives in `version_cmd`
  (CLI layer), not in server.py's `version()` function (which gets overwritten).
- Registry-derived MCP tools call CLI commands via CliRunner. JSON output from CLI
  is properly parsed; non-JSON (old version cmd) returned as raw string. Now fixed.
- Spring DI: constructor-level edge emission (`ClassName#<init>` as `from_symbol`) —
  `_enclosing_class()` correctly extracts class name for endpoint matching.

  security — unchanged by session 40 work.
