## New Findings From This Session

### CRITICAL — P-2: spring-audit hangs on large repos

**`spring-audit` on keycloak (7885 files) produces ZERO output after 180+ seconds.**

- Exit code 0 (no crash), but nothing written to stdout or stderr
- `migrate-check` on same repo completes fine in <60s
- Root cause: `build_canonical_ir()` with full symbol resolution is O(n²) on large repos
- `find_java_files` capped at 8000 but `build_canonical_ir` processes all 5918 non-test files
- Reproduced: `timeout 180 sourcecode spring-audit /Users/user/Documents/workspace/testing/keycloak` → 0 bytes

**Fix direction:** Add progressive timeout + partial results in `build_canonical_ir`, or add `--max-files` cap to spring-audit that emits partial results with limitation note.

### HIGH — P-1: spring-audit performance on monolith

Single class with 8000 endpoints (50k line file) takes **72 seconds**. No crash, correct output, but latency is unacceptable for interactive use.

### MEDIUM — F-1: Java records not parsed by canonical IR

Java `record` types are completely invisible to the canonical IR:
- `@RestController` on record → endpoints NOT detected
- `@Entity` on record → stereotype returns `unknown`
- Records absent from graph nodes entirely
- Verified: `RecordController` with `@GetMapping("/list")` produces 0 endpoints in `project_endpoint_surface()`

**Root cause:** The Java class declaration regex in `build_canonical_ir` / `repository_ir.py` does not match `public record Foo(...)` syntax.

**Fix direction:** Add `record` to class-type parsing regex patterns.

### LOW — A-1: tests/ module exclusion

Multi-module project with a module literally named `tests` has ALL production code silently excluded by `find_java_files`:
```
tests/src/main/java/com/example/IntegrationHelper.java  → EXCLUDED
```
The `rel.startswith("tests/")` pattern is intentional but creates a false negative for this edge case.
