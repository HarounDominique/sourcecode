## Audit of 1.33.7 commit (d7106a3)

Reviewed all 5 fixes from session 38. Found cascade breakage from the
`security: null → {"policy": "none_detected"}` change:

### Root cause
`extract_java_endpoints` (and `project_endpoint_surface` in canonical_ir)
previously emitted `security: null` for unannotated endpoints.
After 1.33.7, they emit `{"policy": "none_detected"}` — a truthy dict.
5 consumers assumed `not e.get("security")` or `e.get("security")` (truthiness)
still worked to distinguish secured vs unsecured endpoints.

### Bugs fixed (commit 72c4a7a → 1.33.9)

1. **`repository_ir.py` L3018** — `no_security_signal` always 0: dict truthy, so
   `not e.get("security")` always False. Fix: `policy == "none_detected"`.

2. **`repository_ir.py` L3037** — `_has_annotation_security` always True: every
   endpoint has a truthy dict now. Fix: `policy != "none_detected"`. Consequence:
   `security_model` was always "annotation_based" or "mixed", never "unknown".

3. **`canonical_ir.py` L443-451** — `project_endpoint_surface` omitted `security`
   key entirely for unsecured endpoints (old behavior: no key = absent). After 1.33.7,
   `extract_java_endpoints` emitted `{"policy": "none_detected"}`, causing schema
   divergence between the two paths. Fix: emit `{"policy": "none_detected"}` in
   `project_endpoint_surface` too. `no_security_signal` count check updated.

4. **`test_canonical_ir.py`** — 3 tests with stale truthiness checks:
   - `test_no_security_signal_count_correct`: `not e.get("security")` → 0==0 vacuous pass
   - `test_spring_security_projection_parity`: counted all endpoints as "secured"
   - `test_security_consistency_no_divergence`: CIR(None) vs projected("none_detected") diverge

5. **`test_enterprise_benchmarks.py`** — 2 tests:
   - `test_roles_allowed_annotation_detected`: `secured` included all endpoints
   - `test_no_security_signal_accurate`: `manually_unsecured` always 0

6. **`mcp/server.py`** — `get_endpoints` docstring said "security dict when
   authorization annotations are present" — stale. Updated: always present.

### Also verified (no bugs)

- `security_surface_affected` in `compute_blast_radius` (L3416): not affected —

<decisions_made>

  `project_endpoint_surface` now emit it. All consumers (production + tests) use
  `policy == "none_detected"` / `policy != "none_detected"` to distinguish.

  CIR to projected output normalize "none_detected" → None.
