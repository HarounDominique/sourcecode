## Architecture reference (key decisions)

- Output schema: `spring_audit_result` with `schema_version:1.0`, findings list, summary stats
- Finding ID: `f"{pattern_id}-{hashlib.sha256(f'{pattern_id}:{symbol}'.encode()).hexdigest()[:12]}"`
- Pattern extensibility: `TxPattern` Protocol ‚Äî add class + register in `_DEFAULT_TX_PATTERNS`
- Security patterns: separate `SecurityPattern` Protocol + `SecurityScanner` class (same structure as TxPatternEngine)
- BFS depth: 3 hops max, 500ms timeout per engine ‚Äî limitations logged in result
- `run_tx_audit(cir, root, scope, min_severity)` + analogous `run_security_audit(cir, root, scope, min_severity)`
