## 1.33.7 ‚Äî 5 contract fixes (commit d7106a3)

- Fix 1 ‚Äî impact filepath resolution: `_resolve_target` extracts basename when
  target is path-like (contains `/` or `\` or ends in `.java`). Path
  `src/main/java/org/foo/Bar.java` ‚Üí strips `.java` ‚Üí extracts basename `Bar`
  ‚Üí suffix-matches to `org.foo.Bar` FQN. Root cause: prior path-to-dot
  conversion (`src.main.java.org.foo.Bar`) never matched any FQN in reverse_graph.
  Ambiguous (multiple `Bar` classes) ‚Üí returns structured
  `{"resolution": "ambiguous_path", "candidates": [...]}`.

- Fix 2 ‚Äî security null eliminated: `entry["security"]` now always emits
  `{"policy": "none_detected"}` instead of `null` when no security signal exists.
  Agents can now test `security.policy` without null-guard. Applies to all
  endpoint entries in `sourcecode endpoints` output.

- Fix 3 ‚Äî review-pr no_diff_source hint: `error_hints` list now populated when
  `no_diff_source` error occurs (no `--since` + no staged changes). CLI emits
  `"hint"` array in error JSON: `["Add --since <ref>...", "Examples: ..."]`.

- Fix 4 ‚Äî get_ir_summary IR access guide: `_ir_access` key added to every
  non-error response from `get_ir_summary` MCP tool. Contains `nodes_path`,
  `edges_path`, and `note` ‚Äî eliminates `d["nodes"]` ‚Üí KeyError pattern in agents.

- Fix 5 ‚Äî version: 1.33.6 ‚Üí 1.33.7 across `__init__.py`, `pyproject.toml`, `README.md`.
