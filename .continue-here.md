## Bugs to fix (priority order)

### HIGH ‚Äî BUG-1 through BUG-4 (single root, fix together)

**Root cause:** `InjectionGraph.build()` in `cir_graphs.py` handles `#` (constructor injection FQN like `Class#<init>`) but NOT `.` (field injection FQN like `Class.fieldName`). All four bugs share this root.

**Impact:** every repo using `@Autowired`/`@Resource` field injection (openmrs-core, BroadleafCommerce, keycloak). Constructor-injection-only repos (petclinic) unaffected.

| Bug | Command | Field | Symptom |
|-----|---------|-------|---------|
| BUG-1 | `explain` | `incoming_callers` | Field names (`dao`, `identifierValidators`) appear as phantom callers |
| BUG-2 | `explain` | `outgoing_deps` | Always empty despite `@Autowired` fields present |
| BUG-3 | `pr-impact` | `modified_classes` | Field FQNs (`PatientServiceImpl.dao`) listed as modified classes |
| BUG-4 | `pr-impact` | `direct_callers` | Field FQNs (`AllergyValidator.patientService`) appear as phantom entries |

**Fix location 1 ‚Äî `cir_graphs.py` `InjectionGraph.build()` (~line 183):**
```python
if "#" in from_fqn:
    class_fqn = from_fqn.rsplit("#", 1)[0]
elif "." in from_fqn:           # ADD: handle field injection dot-notation
    class_fqn = from_fqn.rsplit(".", 1)[0]
else:
    class_fqn = from_fqn
```

**Fix location 2 ‚Äî `pr_impact.py` `_build_file_class_index()` (~line 150):**
```python
kind = node.get("symbol_kind") or node.get("type") or ""
if kind == "field":
    continue
```

**Fix location 3 ‚Äî `spring_impact.py` BFS caller traversal:** filter FQNs not in known class symbol set before adding to callers list.

**Fix location 4 ‚Äî `explain.py` `_build_callers()` (~line 255):** skip FQNs with `.fieldName` suffix (not known class FQNs).
