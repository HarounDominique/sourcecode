### R1 — keycloak "LDAP authentication synchronization failure": CONFIRMED + FIXED
**Observed (before fix):** 8 files, all `symptom_match`, `symptom_explain` dropped, zero LDAP files.

**Root cause chain:**
1. Keywords: `["ldap", "authentication", "synchronization"]`
2. Pass 4 path injection: "authentication" matched 50+ files in keycloak's `authentication/` dirs — no cap.
3. Pass 4c co-location injected all siblings of those dirs — hundreds more files.
4. Total JSON: 1025KB → budget trimmer (200KB cap) cut `relevant_files` to 8 — all generic auth matches.
5. LDAP/sync files (lower score, fewer keyword hits) cut entirely.

**Fix (prepare_context.py, Pass 4 + 4c):**
- Pass 4: per-keyword injection cap `_P4_KW_CAP = 15` in large repos (`_is_large_repo`).
  Files matching multiple keywords exhaust each keyword's quota independently — more specific files survive longer.
- Pass 4c: total co-location cap `_P4C_CAP = 30` in large repos.

**Result after fix:**
- 94 files, no budget trimming, no `_budget_note`.
- LDAP files present: `LDAPStorageProvider.java`, `LDAPConstants.java`, `LdapSettingsSynchronization.tsx`, `SynchronizationResult.java`, `ImportSynchronization.java`, 15+ more.
- Roles: source (21), entrypoint (9), symptom_match (64).
- Keywords: `['ldap', 'authentication', 'synchronization']`.
- Petclinic: 38 files, no regression.

**Also committed:** latent test fix from session 43 — `test_signal_hierarchy.py:533` missing `no_diff_source`.
