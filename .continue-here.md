### Observations (not bugs, not requiring fixes)

1. **`direct_callers` duplication** ‚Äî field injection node (`Class.fieldName`) AND class node (`Class`) both listed as separate callers. Cosmetic/UX issue, inflates caller count. Not a correctness bug.
   - Example: `CheckoutServiceImpl.orderService` + `CheckoutServiceImpl` both in direct_callers of OrderServiceImpl
   - Root cause: CH-002 expansion adds class node; reverse_graph also has field node as separate key
   - Fix value: medium (clarity), risk: low

2. **Security model detection gap** ‚Äî all 8 repos return `security_model=unknown`. None use Java annotation-based `@EnableWebSecurity`. BLC uses XML-based Spring Security. This is a structural limitation (XML not parseable), not a bug.

3. **TX-002/004 structural gap** ‚Äî openmrs has `NOT_SUPPORTED` + `REQUIRES_NEW` methods on class with class-level `@Transactional(REQUIRED)`, but TX-002/004 don't fire because self-invocation and call chains are not in static call graph. Documented limitation, not a bug.

4. **analysis_time_ms in output JSON** ‚Äî volatile field included in output; hash-based CI scripts must exclude it. Low priority documentation/schema issue.
