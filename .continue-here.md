## Session 46 — 3-Fix Parent-Git Audit

**Objective**: Fix 3 bugs triggered when `.git` is in a parent directory (e.g. MSAS/.git
with saint-server/ as the analyzed subdirectory).

### BUG-1 (CRITICAL) — Cache never written for parent-git repos (cli.py)
**Symptom**: `cache_source: "fresh"` on every call, `cache warm` reports success but writes nothing.
**Root cause**: `cli.py:1146` had `(target / ".git").exists()` — False when `.git` is in a parent.
`_core_key` stayed `""`, blocking both L1/L2 write and RIS update.
**Fix**:
1. Added `git rev-parse --show-toplevel` detection to find the actual git root regardless of depth.
2. Changed cache condition from `(target / ".git").exists()` to `_git_root_str` (non-empty = git available).
3. Changed RIS update condition from `_core_key` to `_git_sha` so RIS is always written when git is available.
**Commit**: `4deb198`

### BUG-2 (HIGH) — Path prefix inconsistency in fix-bug output (prepare_context.py)
**Symptom**: 2 of 36 paths had `saint-server/` prefix; agents got ENOENT opening them.
**Root cause**: Pass 3 commit-file injection used raw `files_changed` from git log, which are
relative to the git root (MSAS/). Paths like `saint-server/src/...` were injected directly.
**Fix**: Strip `self.root.name + "/"` prefix from each commit file path before adding to
`_commit_file_hits`, normalizing them to be relative to the analyzed directory.
**Commit**: `4deb198`

### BUG-3 (HIGH) — MCP check_freshness returns current_git_head: "" (server.py)
**Symptom**: MCP returns `{"current_git_head": "", "ris_git_head": "", "fresh": false}`;
CLI correctly shows `Current HEAD: ab71b2bf`.
**Root cause**: `_execute(["cache", "freshness", repo_path, "--json"])` went through CliRunner
which caused subprocess nesting issues on Windows with parent-git repos.
**Fix**: Replaced `_execute(...)` with direct Python calls to `_get_git_head / load_ris /
_has_uncommitted_changes`, replicating `cache_freshness_cmd` logic without CliRunner.
**Commit**: `4deb198`

<decisions_made>

  handles the common case without needing to compute git root in prepare_context.
