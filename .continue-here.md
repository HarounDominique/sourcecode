### NEW BUG FOUND AND FIXED — MCP get_compact_context/get_agent_context wrong repo
**Symptom**: get_compact_context(repo_path="/path/to/spring-petclinic") returned
atlas-cli project context (project_type='cli') instead of spring-petclinic (web_mvc).
**Root cause**: `runner.py:run_command()` called `_set_detected_path(".")` then
`_preprocess_args(args)` (strips path, stores in TLS). Then `_runner.invoke(app, processed)`
was called with path-stripped args. Inside invoke, `_cmd_main` in cli.py ALSO calls
`_set_detected_path(".")` + `_preprocess_args` — but on the already-stripped args, so
the path was "." (current dir = atlas-cli). This affected ALL registry-dispatched MCP
tools: get_compact_context, get_agent_context, fix_bug_context, etc.
**Fix**: Removed `_set_detected_path(".")` + `_preprocess_args()` from `runner.py`.
Now passes raw args to `_runner.invoke`; `_cmd_main` handles path extraction once.
**File**: `src/sourcecode/mcp/runner.py`
