## Bugs Found and Fixed

### IC-V1 — Duplicate seed FQNs
- **Root cause**: CIR has duplicate symbols (e.g., `OwnerController#addPaginationModel` ×2). `_resolve_symbol` didn't deduplicate.
- **Fix**: `list(dict.fromkeys(...))` in `_resolve_symbol` output.
- **Regression test**: `TestRegressionV1DeduplicateSeeds`

### IC-V2 — `resolved_symbol` = short input name
- **Root cause**: `resolved_symbol = seed_fqns[0] if len(seed_fqns) == 1 else symbol` — multi-seed class expansion kept original input.
- **Fix**: Extract canonical class FQN when all seeds share one class.
- **Regression test**: `TestRegressionV2ResolvedSymbol`

### IC-V3 — `class_expanded` resolution never emitted
- **Root cause**: Suffix-match class queries returned `"partial"` → `confidence="medium"` unfairly.
- **Fix**: New `"class_expanded"` resolution for unambiguous suffix match. Confidence stays `"high"`.
- **Regression test**: `TestRegressionV3ClassExpandedConfidence`

### IC-V5 — Endpoint collection over-includes
- **Root cause**: `_collect_endpoints` included ALL endpoints of any controller whose class appeared in call chain. Single-method query returned 7 endpoints (should be 1).
- **Confirmed on**: petclinic `OwnerController#processCreationForm` (7→1), Broadleaf `RegisterCustomerController#registerCustomer` (2→1)
- **Fix**: Include endpoint only if `handler_symbol in all_fqns` OR controller is a class-level seed.
- **Regression tests**: `TestRegressionV5EndpointPrecision` (3 tests)

### IC-V6 — `imports` edges pollute call chain
- **Root cause**: `imports` = type reference (A declares field/param of type B), not a call. BFS traversed them, creating false call chains up to 4 hops through DTOs/entities. OrderService had 270 callers (40+ via imports), all non-callers.
- **Evidence**: AdminProductController appeared in OrderService chain via: `OrderService ← [imports] OfferService ← [imports] OrderServiceImpl ← [imports] AdminProductController` — a 4-hop type-ref chain.
- **Fix**: Added `"imports"` to `_SKIP_EDGE_TYPES` (alongside `"contained_in"`).
- **Regression tests**: `TestRegressionImportsEdgeExclusion` (3 tests)

## Known Limitations (post-fix, documented)

1. **JPA/Spring Data interface callers**: `OwnerRepository#findById` has 0 callers in CIR (interface methods without concrete implementation in call graph). Direct call chains to/from Spring Data repositories are not tracked.
2. **`implements` edges missing for large service interfaces**: `OrderService → OrderServiceImpl` `implements` edge not in CIR for Broadleaf. TX findings on implementation methods won't appear in impact chain for interface query.
3. **`injects` callers are field nodes**: `CheckoutServiceImpl.orderService` (field FQN), not method FQNs. Field nodes are leaf nodes — indirect traversal stops at depth=1 for inject-only dependencies.
4. **SEC findings = 0 for Broadleaf** (correct by design — `security_model=unknown` for filter-based repos, SEC-001 not emitted)
5. **Self-invocation** — pre-existing, needs tree-sitter
