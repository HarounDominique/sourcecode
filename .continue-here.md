### Fix 1: file_relevance Spring controller differentiation (v1.35.10)
**Commit:** `5794961`

**Problem:** All Spring controllers collapsed to `runtime_core / 0.95` in `file_relevance`.
Bootstrap class (`@SpringBootApplication`) and `@RestController` were indistinguishable.

**Root cause:** `FileClassifier.classify()` had early-return for `production_entry_paths`
BEFORE Java stereotype detection. All Spring controllers are declared entry points,
so they all got `runtime_core/0.95` regardless of annotation type.

**Fix in `src/sourcecode/file_classifier.py`:**
- Extended `_JAVA_ANNOTATION_RE` with HTTP mapping annotations
- Added `_HANDLER_MAPPING_RE` for method-level handler counting
- Added 6 `@Controller+{Mapping}` rules at base 0.80 in `_JAVA_STEREOTYPE_RULES`
- Added `@Controller` alone at 0.72
- Moved stereotype detection BEFORE `production_entry_paths` fallback
- Added handler-count boost (`+0.03/handler, max +0.12`) for `@Controller` tier only
  (guard: `relevance < 0.85` — skips `@RestController` at 0.90)
- Added `_count_handlers()` method

**Fix in `src/sourcecode/serializer.py`:**
- Extended T1 score override to `api_endpoint`/`security`/`exception_handler` categories
  in entry_paths (min 0.70, was previously only `runtime_core`/`cli_entrypoint`)

**Verified on spring-petclinic (fresh run):**

0.920 [api_endpoint]  PetController.java          (5 handlers)

### Fix 2: architecture.confidence inconsistency (v1.35.11)
**Commit:** `5c38ca2`

**Problem:** When architecture analyzer detects a real pattern (e.g. `spring_mvc_layered`)
but emits `confidence="low"` due to missing docs (no OpenAPI/ADR), the serializer
emitted that raw "low" in:
- `architecture.confidence` field
- `confidence_summary.sections.architecture`

But `confidence_analyzer.py` already promoted low→medium when pattern was detected,
so the factor message said `"architecture.confidence=medium"` while the field said "low".
Contradictory signal for agents.

**Fix in `src/sourcecode/serializer.py`:**
- `_architecture_context()`: compute `_eff_confidence = "medium" if arch.confidence == "low" and pattern else arch.confidence`
- `_section_confidence()`: same promotion for `sections.architecture`

**Verified on petclinic:**
```
architecture.confidence: medium  ✓
sections.architecture: medium    ✓
factors: ["architecture.confidence=medium (pattern=spring_mvc_layered) → ..."]  ✓
```
