- **Files:** `repository_ir.py`, `spring_semantic.py`, `spring_model.py`, `test_spring_semantic.py`, `test_spring_model.py`

### BUG-5 FIXED: chunk-file misclassifies multi-line method signatures
- **File:** `src/sourcecode/file_chunker.py`
- **Cause (2 sub-issues):**
  1. `_is_method_or_constructor_start` only fired when `{` was on same line as method name. Multi-line signatures (e.g., `public String saveEntity(\n  Req req,\n  ...) throws Exception {`) fell into `field_block` — producing 592 and 1159 line chunks.
  2. `_METHOD_RE` didn't match `public @ResponseBody Map<String,Object> method(` — return-type annotations (`@ResponseBody`, `@Nullable`, etc.) between modifiers and return type caused regex to fail.
- **Fix:**
  1. `_pending_method` state: when method name detected without `{`, record it; complete on subsequent line where `opens > closes` (net brace increase). Guard avoids false completion on `@RequestMapping({"v1","v2"})`.
  2. `_METHOD_RE`: added `(?:@\w+(?:\([^)]*\))?\s+)*` group after modifiers to absorb return-type annotations.
- **Result:** AdminBasicEntityController (118KB): 35 → 107 chunks, 61 methods, 0 large field_blocks
