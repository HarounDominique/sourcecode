## Session 42 ŌĆö Graph Expansion + Budget Fix + Schema Compat (commit 991409e ŌåÆ 1.33.13)

### Primary Bug
`fix-bug --symptom` returned exactly 3 files on large repos (Keycloak ~6758 classes).

Root causes:
1. **Budget collapse**: `output_budget.py` Step 6 (last resort) trimmed `relevant_files` ŌåÆ 3
   when JSON exceeded 100KB (`BUDGET_FIX_BUG = 100_000`).
   - Fix: `BUDGET_FIX_BUG` 100K ŌåÆ **200K**; last-resort cap 3 ŌåÆ **10**

2. **No graph expansion**: Pipeline only did path/commit matching + directory co-location.
   Files like `JpaUserProvider.java` (implements `UserProvider`) were never found because
   they don't contain symptom keywords in their own path.
   - Fix: **Pass 5 (reverse-import lookup)** ŌĆö for each high-score seed class (score Ōēź 0.5),
     scans candidate source files for word-boundary references (imports/implements/extends).
     Small repos: scans all non-test source files (fresh reads up to 8KB each).
     Large repos: uses `_scanned_body` cache only (no additional I/O).
     1-hop neighbors added with decayed score = seed ├Ś 0.6 (capped at 0.85).

3. **Silent expansion**: `explanation` field never mentioned graph expansion.
   - Fix: `why = "graph_expansion: 1 hop from {stem}"`; `symptom_explain.graph_expansion` list.

<decisions_made>

- Pass 5 uses **reverse-import** (who imports the seed?), not forward-import (what does
  the seed import?). Forward lookup finds model/DTO deps ŌĆö less useful for debugging.
  Reverse finds implementations and consumers ŌĆö exactly what fix-bug needs.
- `_scanned_body` dict caches raw content during the content-scan loop so Pass 5 can
  reuse already-read files without extra I/O.
- Small repos (Ōēż 500 files): Pass 5 scans all non-test source files. Large repos: Pass 5
  uses `_scanned_body` only ŌĆö avoids O(n) file reads on Keycloak-scale repos.
- `_gx_seed_stems` collects from ALL high-score files (not just `role="symptom_match"`),
  because files found by `_rank_files` class-name boost have `role="source"` but are
  equally valid seeds.
- Budget: `BUDGET_FIX_BUG` 100KŌåÆ200K is conservative enough for most outputs; the
  last-resort 3ŌåÆ10 is the safety net. Raising to 200K means step 6 only fires for
  truly exceptional outputs (>200KB JSON).
- `file`/`path` dual-emit is one-release only. Remove the `d["file"] = d["path"]` line
  in 1.33.14 or 1.34.x after downstream consumers have updated.
