## Bugs Found and Fixed

### BUG-005 — TX-005 Mass False Positives (HIGH) ✓
**File**: `src/sourcecode/spring_tx_analyzer.py`
**Root cause**: `_has_swallowed_exception` scanned entire file, not specific method body.
One swallowed catch in any method flagged ALL TX methods in the file.
**Impact**: 32 of 36 TX-005 findings on BroadleafCommerce were false positives.
**Fix**: Added `_extract_method_body()` — brace-counting body extractor. Scans only target method.
**After**: 36 → 4 TX-005 findings (all legitimate)
**Tests**: +2 in TestTX005

### BUG-003 — Event Publisher Javadoc False Positives (HIGH) ✓
**File**: `src/sourcecode/repository_ir.py`
**Root cause**: `_PUBLISH_EVENT_RE` applied to raw source including Javadoc `/** ... */`.
BroadleafSystemEvent.java Javadoc example generated 3 false publisher edges.
**Fix**: Added `_strip_java_comments()`. Applied before event regex scan.
**After**: 0 false edges; real edges unchanged
**Tests**: +2 in TestBUG003JavadocFalsePositive

### BUG-004 — Event FQN Mismatch Same-Package Events (MEDIUM) ✓
**File**: `src/sourcecode/repository_ir.py`
**Root cause**: Same-package events not in import_map → stored as simple name → FQN lookup misses.
**Fix**: Resolution now tries `import_map → same_pkg_types → simple name`
**After**: CustomerPersistedEvent topology returns 4 publishers (was 0)

### BUG-002 — Null tx_boundary for Class with Method-Level TX (MEDIUM) ✓
**File**: `src/sourcecode/spring_impact.py`
**Root cause**: Class query checked `class_level` only, not `by_class`.
**Fix**: Falls back to first entry in `by_class[class_fqn]` when class_level is empty.
**After**: StaticAssetServiceImpl class query shows `transaction_boundary: {scope: method}`
**Tests**: +1 in TestBUG002TxBoundaryClassLevelQuery
