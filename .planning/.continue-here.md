**Session 95 — Output Pipeline Unification:**

- **Audit** — identified two parallel output systems coexisting without coordination:
  - System A: `write_output()` in serializer.py (used only by `main()`) — own file error handler, no clipboard
  - System B: inline `sys.stdout.buffer.write` + `_safe_write_file` blocks in ~15 command functions
  - Root cause: no unified emit function → divergent error envelopes, inconsistent clipboard behavior

- **Refactoring** — `_emit_command_output(content, output_path, copy, *, success_msg=None)`:
  - Single function handles stdout / `--output` file / `--copy` clipboard for all commands
  - File write failures route through `_safe_write_file` → unified `INVALID_INPUT_CODE` JSON on stderr
  - UnicodeEncodeError + AttributeError fallbacks consolidated in one place

- **Fixes included:**
  - L1 cache inline ruamel.yaml block (last surviving H1 item) → `_serialize_dict`
  - `impact_cmd`: added `--format yaml` support (was JSON-only), uses `_serialize_dict`
  - `rename_class_cmd`, `chunk_file_cmd`: `to_yaml + json.dumps` → `_serialize_dict`
  - 11 commands × 7-line emit blocks → single `_emit_command_output` call each
  - `write_output()` in serializer.py: deprecated (body kept for external callers)
