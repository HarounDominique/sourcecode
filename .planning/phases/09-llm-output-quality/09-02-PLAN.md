Reglas de inferencia de importance (D-03 de CONTEXT.md):
```
_infer_importance(path, kind, entry_points) -> Literal["high","medium","low"]:
  depth_count = path.count("/")   # posix path: "src/main.py" -> 1, "src/core/base.py" -> 2

  if entry_points and path in entry_points:
      return "high"
  if depth_count <= 0:           # modulo en raiz: "main.py"
      return "high"
  if depth_count == 1:           # modulo a un nivel: "src/main.py"
      return "high"
  if depth_count == 2 or kind in {"class", "function"}:
      return "medium"
  return "low"
```
Nota: la profundidad se cuenta como numero de "/" en el path relativo posix.
depth=1 significa exactamente un directorio de profundidad (ej. "src/main.py").
Un archivo en la raiz como "main.py" tiene depth=0 -> high.

```python
@staticmethod
def _infer_importance(
    path: str,
    kind: str,
    entry_points: list[str] | None,
) -> Literal["high", "medium", "low"]:
    """Infiere importancia desde senales estructurales."""
    if entry_points and path in entry_points:
        return "high"
    depth = path.count("/")
    if depth <= 1:   # raiz (0) o un nivel (1): src/main.py
        return "high"
    if depth == 2 or kind in {"class", "function"}:
        return "medium"
    return "low"
```
